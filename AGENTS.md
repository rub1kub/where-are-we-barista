# Инструкции для AI-агентов

Проект: **КРОТ** — Корреляционный Рельефный Определитель Траектории.  
Команда: **«Где мы, Бариста?»**.

## Контекст

Это хакатонный прототип для кейса «Полёт вслепую»: воздушное судно теряет ГНСС, но должно восстановить координаты, путевую скорость и азимут по данным радиовысотомера и цифровой модели рельефа.

Сценарий демонстрации строго гражданский: доставка груза в удалённый район тайги, тундры, степи, горного плато или геологоразведочного лагеря.

## Ключевые правила продукта

- Главный экран по умолчанию — операторская/инженерная панель, а не учебный лендинг.
- Пояснения для трекеров и жюри держать в отдельном режиме «Методика».
- В основном режиме показывать только рабочие данные: ГНСС, БАРО, РВ, ЦМР, координаты WGS-84, Vпут, азимут, corr, СКО, достоверность, NMEA, профиль, поверхность корреляции, карту и 3D-превью.
- Аббревиатуры из ТЗ можно сохранять: ГНСС, NMEA, ЦМР, БАРО, РВ, MSL, AGL, WGS-84, corr, СКО.
- Англицизмы в видимом интерфейсе заменять русскими терминами, если это не стандартная аббревиатура.
- UI должен быть квадратным, тёмным, плотным, техническим. Не использовать карточный маркетинговый стиль.
- Не добавлять военный сценарий, наведение, поражение, тактические рекомендации или другую не относящуюся к гражданскому кейсу функциональность.

## Запрещённые фразы на основном экране

Не возвращать в операторский режим эти формулировки:

- «простыми словами»
- «почему ответ выглядит надёжно»
- «борт меряет землю»
- «рисунок рельефа»
- «можно использовать»
- «что реально происходит»
- «куда летим»
- «где мы сейчас»

Если нужен учебный текст, размещать его только в режиме «Методика» или README.

## Архитектура

- `src/terrainMatcher.ts` — ядро алгоритма: NMEA → профиль рельефа → перебор азимута/скорости/смещения → корреляция → оценка координаты.
- `src/copernicusDemSample.ts` — сэмпл реальной ЦМР Copernicus DEM GLO-30 для района Ванавары.
- `scripts/generate-dem-sample.mjs` — воспроизводимый генератор сэмпла из открытых COG-тайлов Copernicus DEM.
- `src/nmeaRadioAltimeter.ts` — генерация и разбор NMEA-0183 сообщений радиовысотомера.
- `src/App.tsx` — операторский интерфейс, карта, вводные, режим «Методика».
- `src/FlightPreview3D.tsx` — WebGL/Three.js превью БВС над тайгой.
- `src/styles.css` — тёмная квадратная техническая тема.
- `docs/ai/repository-knowledge-base.json` — машинно-читаемая база знаний для AI-инструментов.

## Команды

```bash
npm install
npm run dev
npm test
npm run build
npm run dem:generate
```

`npm run dem:generate` требует сеть и перегенерирует `src/copernicusDemSample.ts` из публичных Copernicus DEM COG.

## Критерий готовности изменений

Перед сдачей изменений запускать:

```bash
npm test
npm run build
```

Для визуальных правок дополнительно открыть `http://127.0.0.1:5173/` и проверить:

- главный экран не пустой;
- нет framework overlay;
- в консоли нет app error/warn;
- режим «Оператор» открыт по умолчанию;
- режим «Методика» переключается;
- 3D canvas видим и не чёрный;
- карта маршрута и линии траекторий видимы.
