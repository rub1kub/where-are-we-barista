# Где мы, Бариста?

Прототип для хакатон-кейса **«Полёт вслепую»**: воздушное судно теряет спутниковую навигацию, но продолжает определять координату, скорость и направление по радиовысотомеру и цифровой карте рельефа.

Проект сделан как демонстрационный операторский стенд для трекеров и жюри: слева видны входные данные, в центре карта маршрута, справа — понятный ответ «где мы сейчас» и насколько этому ответу можно доверять.

## Коротко

- Название проекта: **Где мы, Бариста?**
- Сценарий: гражданская доставка груза в удалённый район.
- Район: Красноярский край, средняя тайга, маршрут **Ванавара → геологоразведочный лагерь**.
- Спутниковая навигация: считается недоступной.
- Основные входы: барометр, радиовысотомер NMEA-0183, цифровая модель рельефа.
- Основной метод: корреляция измеренного профиля рельефа с эталонными профилями карты.

## Почему район не Украина

Для демонстрации тайги Украина не подходит как основной район: её природные зоны описываются как лесная, лесостепная, степная и средиземноморская зоны, без бореальной тайги.

Для кейса с тайгой выбран район Ванавары в Красноярском крае. Тайга как биом связана с северными бореальными лесами, особенно Сибирью, а река Подкаменная Тунгуска проходит через Красноярский край и район Ванавары.

Источники для обоснования района:

- Encyclopedia of Ukraine: vegetation zones of Ukraine — https://www.encyclopediaofukraine.com/display.asp?linkpath=pages%5CF%5CL%5CFlora.htm
- Britannica: Taiga — https://www.britannica.com/science/taiga
- Britannica: Podkamennaya Tunguska River — https://www.britannica.com/place/Podkamennaya-Tunguska-River

## Что видно в интерфейсе

- **Что поступает на вход**: карта рельефа, барометр, радиовысотомер, диапазон скорости.
- **Вводные стенда**: скорость, направление, длительность участка, шум датчика.
- **Карта маршрута**: спутниковая подложка, фактический путь, найденный путь.
- **Где мы сейчас**: координаты WGS-84, скорость, направление, доверие к оценке.
- **Поток NMEA**: последние сообщения радиовысотомера.
- **Профиль рельефа**: измеренный профиль и профиль карты.
- **Карта совпадений**: где алгоритм нашёл максимальную корреляцию.
- **Превью полёта**: компактный технический вид борта над рельефом.

## Как это решает задачу кейса

1. Борт летит на известной абсолютной высоте по барометру.
2. Радиовысотомер показывает расстояние от борта до земли.
3. Вычисляется профиль абсолютных высот земли:

```text
высота_рельефа_MSL = высота_барометра_MSL - радиовысота_AGL
```

4. Алгоритм строит эталонные профили по цифровой карте рельефа.
5. Перебираются азимуты 0-359 градусов и диапазон скоростей.
6. Для каждого варианта считается корреляция с измеренным профилем.
7. Максимум корреляции даёт координату, путевую скорость и путевой угол.
8. По выраженности рельефа, RMSE и пику корреляции считается доверие к координате.

## Что важно сказать жюри

Это не «ИИ угадывает координату». Прототип решает прикладную навигационную задачу через проверяемую физику:

- радиовысотомер даёт расстояние до земли;
- барометр даёт абсолютную высоту борта;
- разность даёт профиль рельефа под трассой;
- профиль рельефа сравнивается с картой;
- лучшее совпадение даёт координату и направление движения.

Такой подход полезен именно там, где нет плотной наземной инфраструктуры: тайга, тундра, степь, горные плато, удалённые базы и лагеря.

## Запуск

```bash
npm install
npm run dev
```

Открыть локальный URL из вывода сервера.

## Проверки

```bash
npm test
npm run build
```

Smoke-тесты проверяют:

- разбор и генерацию NMEA;
- контрольную сумму NMEA;
- геопривязку маршрута в районе Ванавары;
- восстановление азимута и скорости;
- построение тепловой карты;
- снижение доверия на плоском шумном рельефе.

## Что можно улучшить после MVP

1. Подключить реальную DEM-плитку SRTM/Copernicus/ALOS вместо синтетической функции высот.
2. Расширить компактное превью полёта до отдельного 3D-режима с камерой, высотой над землёй и текущей точкой профиля.
3. Добавить фильтр Калмана для сглаживания радиовысотомера.
4. Добавить несколько районов: тайга, тундра, степь, горное плато.
5. Добавить импорт реального NMEA-журнала.
6. Оптимизировать расчёт под маломощный бортовой вычислитель через WebWorker/WASM.
7. Добавить экспорт отчёта для постфактум-анализа полёта.

## Ограничения текущего прототипа

- Спутниковая подложка реальная, но цифровая модель высот пока синтетическая.
- Реальный бортовой журнал не импортируется через UI, данные генерируются стендом.
- Превью полёта в MVP компактное; полноценный 3D-режим лучше делать отдельной вкладкой, чтобы не перегружать первый экран.
