# КРОТ

**КРОТ** — Корреляционный Рельефный Определитель Траектории.  
Проект команды **«Где мы, Бариста?»** для хакатон-кейса **«Полёт вслепую»**.

Прототип восстанавливает координаты, путевую скорость и азимут воздушного судна при недоступной ГНСС. Входы: БАРО, радиовысотомер в формате NMEA-0183 и цифровая модель рельефа.

## Сценарий

- Гражданская доставка груза в удалённый район.
- Район: Красноярский край, средняя тайга.
- Маршрут: **Ванавара → геологоразведочный лагерь**.
- ГНСС: недоступна.
- Бортовые источники: БАРО MSL, РВ AGL, NMEA-0183, ЦМР.
- ЦМР: сэмпл реального **Copernicus DEM GLO-30 COG**.

## Почему тайга не в Украине

Для демонстрации тайги выбран Красноярский край. Украина описывается через лесную, лесостепную, степную и средиземноморскую зоны, без бореальной тайги. Тайга как биом характерна для северных бореальных лесов, в том числе Сибири.

Источники:

- Encyclopedia of Ukraine: vegetation zones of Ukraine — https://www.encyclopediaofukraine.com/display.asp?linkpath=pages%5CF%5CL%5CFlora.htm
- Britannica: Taiga — https://www.britannica.com/science/taiga
- Britannica: Podkamennaya Tunguska River — https://www.britannica.com/place/Podkamennaya-Tunguska-River

## Как работает алгоритм

1. БАРО даёт абсолютную высоту борта над уровнем моря: MSL.
2. Радиовысотомер даёт расстояние до поверхности: AGL.
3. Из этого строится профиль рельефа:

```text
рельеф MSL = БАРО MSL - РВ AGL
```

4. По ЦМР строятся эталонные профили.
5. Азимут перебирается от 0 до 359 градусов с шагом 1 градус.
6. Скорость перебирается в заданном диапазоне.
7. Для каждого варианта считаются corr и СКО.
8. Максимум corr даёт координату WGS-84, Vпут и азимут.
9. Достоверность считается по corr, СКО, выраженности рельефа и неоднозначности пика.

## Что видно в интерфейсе

- **Оператор**: рабочий режим по умолчанию.
- **Журнал NMEA**: постфактум-анализ вставленного или загруженного `.txt/.nmea` журнала радиовысотомера без передачи truth-траектории solver-у.
- **3D-превью БВС**: полёт самолётного беспилотника над тайгой на WebGL/Three.js, ЦМР Copernicus со сглаживанием и спутниковая подложка ArcGIS World Imagery с резервной текстурой без сети.
- **Карта маршрута**: спутниковая подложка, истинная траектория и оценка алгоритма.
- **Оценка положения**: координаты WGS-84, Vпут, азимут, СКО и достоверность.
- **AUTOPILOT OUTPUT**: структурированный выход для внешнего контура: lat/lon, `ground_speed_mps`, `azimuth_deg`, `confidence`, `uncertainty_m`, `course_correction_deg`.
- **События алгоритма**: журнал текущего расчёта от `RA_STREAM_STARTED` до итогового статуса.
- **Поток NMEA**: последние сообщения радиовысотомера.
- **Профиль рельефа**: измерение РВ против эталона ЦМР.
- **Поверхность кандидатов**: тепловая карта corr по азимуту и скорости.
- **Методика**: отдельный режим для объяснения трекерам и жюри.

## DEM-данные

Сэмпл ЦМР генерируется из публичных COG-тайлов Copernicus DEM GLO-30:

```bash
npm run dem:generate
```

Скрипт `scripts/generate-dem-sample.mjs` скачивает тайлы E102-E105 для широтного пояса N60 и создаёт `src/copernicusDemSample.ts`.

Текущий сэмпл:

- сетка: 420×180;
- границы: 60.25-60.95 N, 102.15-105.65 E;
- диапазон высот: 242.6-591.8 м.

## Запуск

```bash
npm install
npm run dev
```

Локальный URL: `http://127.0.0.1:5173/`.

## Проверки

```bash
npm test
npm run build
```

Smoke-тесты проверяют:

- генерацию и разбор NMEA;
- контрольную сумму NMEA;
- геопривязку маршрута в районе Ванавары;
- восстановление азимута и скорости;
- построение heatmap/corr;
- снижение доверия на плоском шумном рельефе;
- работу solver-а без truthPath;
- события расчёта и структурированный AUTOPILOT OUTPUT;
- отказ `NO FIX` на заведомо несовместимом профиле.
