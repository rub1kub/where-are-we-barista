# Финальный сценарий демо КРОТ

## Главная фраза

Мы не определяем координаты нейросетью. Мы строим профиль рельефа по радиовысотомеру и ищем его совпадение на карте высот.

## Порядок показа

1. Открыть операторский экран: `npm run dev`, затем `http://127.0.0.1:5173/`.
2. Показать входы: РВ NMEA, высота `1500 м MSL`, карта высот Copernicus DEM GLO-30.
3. Показать карту в режиме `Карта высот`: цветовая шкала, тени, контуры, текущая траектория.
4. Показать профиль рельефа: измеренный профиль РВ и эталон карты высот.
5. Показать тепловую карту совпадений: перебор азимута и скорости, лучший пик, второй пик.
6. Показать `Выход алгоритма`: координаты X/Y, WGS-84, Vпут, азимут, доверие к расчёту, статус.
7. Переключиться в `Проверочный журнал`.
8. Нажать `Контроль Ванавара` и `Рассчитать по журналу`.
9. Обратить внимание: расчёт идёт от файла NMEA без истинной траектории на входе.
10. Нажать `PX4 пример` и показать честный отказ: координата не выдана, остаётся только диагностический кандидат.

## Команды перед защитой

```bash

```

## Что говорить

- Вход расчёта: журнал радиовысотомера, высота `1500 м MSL`, карта высот района.
- Высота земли считается как `1500 м MSL - РВ AGL`.
- Скорость и азимут не берутся из истинной траектории: они находятся перебором по максимуму совпадения профилей.
- Успешный файл Ванавары — контрольный внешний NMEA для стенда, не данные заказчика.
- PX4-derived файл нужен как отрицательный пример: если профиль не совпадает с картой, система не выдумывает координату.
- 3D-превью показывает текущую оценённую траекторию и высоту над рельефом; расчёт делает не 3D-сцена, а корреляционный алгоритм.

## Что не говорить

- Не говорить, что КРОТ заменяет автопилот.
- Не обещать серийную бортовую интеграцию без доработок.
- Не говорить, что координата находится нейросетью.
- Не называть контрольный журнал данными заказчика.
- Не обещать результат на любом рельефе: на слабом рельефе должен быть `LOW RELIEF`, `FIX AMBIGUOUS` или `NO FIX`.

## Опасные вопросы

**Вы сами сгенерировали успешный журнал?**  
Да, это контрольный внешний журнал для воспроизводимой проверки до получения файла от организаторов. Расчётная часть не получает истинную траекторию, только NMEA, высоту 1500 м и карту высот.

**Почему отрицательный файл не даёт координату?**  
Потому что профиль не совпал с текущей картой высот с достаточным качеством. Это правильное поведение: система не должна уверенно выдавать ложную координату.

**Что означает `fix_usable`?**  
Это признак, что координату можно использовать как навигационную оценку. При `NO FIX`, `LOW RELIEF` и `FIX AMBIGUOUS` координата не выдаётся, а лучший кандидат остаётся только диагностикой.

**Что нужно для реального борта?**  
Нужен согласованный формат журнала РВ, подготовленная карта высот района, проверка шума/выбросов, интеграция как onboard service или companion computer и испытания на реальных данных.

## Финальный критерий готовности

- `npm test` проходит.
- `npm run build` проходит.
- Успешный NMEA Ванавары даёт `FIX VALID` или `FIX DEGRADED`.
- PX4-derived NMEA даёт отказ или непригодный статус без навигационной координаты.
- В README и документах явно написано, что успешный журнал не является данными заказчика.
