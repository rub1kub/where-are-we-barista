# КРОТ как бортовой навигационный модуль

Документ описывает перенос MVP из программного стенда в гражданский бортовой контур лёгкого БВС или опционально-пилотируемого воздушного судна.

## Роль модуля

КРОТ не заменяет автопилот. Модуль работает как дополнительный источник навигационной оценки при недоступной или недостоверной ГНСС.

```text
РВ AGL / NMEA ─────┐
БАРО MSL ──────────┤
ЦМР района ────────┤
                   ↓
        terrain_nav_service
                   ↓
 local_x_m / local_y_m + lat/lon + Vпут + азимут + достоверность
                   ↓
   автопилот / операторский монитор / контур коррекции
```

## Входы

| Вход | Формат | Частота | Назначение |
| --- | --- | ---: | --- |
| Радиовысотомер | NMEA-0183 GGA | 1-10 Гц | Расстояние от борта до поверхности, AGL |
| Барометрическая высота | MSL, м | 1-10 Гц | Абсолютная высота борта |
| ЦМР | Copernicus GLO-30 / SRTM / ALOS | заранее загружена | Эталонный профиль рельефа |
| Параметры поиска | диапазон скорости, шаг, окно | перед расчётом | Ограничения корреляционного поиска |

## Выходы

| Поле | Что значит | Куда передаётся |
| --- | --- | --- |
| `local_x_m` / `local_y_m` | локальные координаты на карте ЦМР, м | автопилот, монитор, журнал |
| `lat` / `lon` | оценка координат WGS-84 | монитор, автопилот, журнал |
| `ground_speed_mps` | путевая скорость | навигационный монитор |
| `azimuth_deg` | путевой угол | контур коррекции курса |
| `confidence` | достоверность оценки | логика принятия решения |
| `uncertainty_m` | оценка радиуса ошибки | контроль безопасности |
| `navigation_status` | `FIX VALID`, `FIX DEGRADED`, `FIX AMBIGUOUS`, `LOW RELIEF`, `NO FIX` | операторский статус |
| `course_correction_deg` | рекомендательная поправка курса к плановой линии, если статус позволяет | автопилот или оператор |

## Режимы интеграции

### Рекомендательный режим

Первый реалистичный режим для опытной эксплуатации. КРОТ считает положение и рекомендацию, но не управляет самолётом напрямую.

```json
{
  "local_x_m": 158820,
  "local_y_m": 54026,
  "lat": 60.829960,
  "lon": 105.488265,
  "ground_speed_mps": 44.0,
  "azimuth_deg": 73,
  "confidence": 0.90,
  "uncertainty_m": 42,
  "navigation_status": "FIX VALID",
  "course_correction_deg": 0.0
}
```

`course_correction_deg` рассчитывается как разница между текущим оценённым азимутом и направлением на плановую линию маршрута через точку упреждения. Если статус `NO FIX`, `LOW RELIEF` или `FIX AMBIGUOUS`, поле остаётся `null`: система не выдаёт совет по курсу при ненадёжной навигационной оценке.

### External navigation source

Следующий этап: передавать оценку как внешний источник положения через serial, CAN, MAVLink или внутренний API бортовой системы. Конкретный транспорт выбирается под автопилот и требования сертификации.

## Что реализовано в MVP

- Парсинг NMEA-0183 радиовысотомера.
- Преобразование `terrain MSL = BARO MSL - RA AGL`.
- Корреляционный поиск по ЦМР.
- Оценка координат, Vпут, азимута, corr, СКО и confidence.
- Статусы качества навигации.
- Постфактум-анализ вставленного NMEA-журнала.

## Что не является частью MVP

- Управление рулями, тягой и стабилизацией.
- Замена автопилота.
- Failsafe-логика полётного контроллера.
- Серийная интеграция с конкретным бортовым протоколом.
