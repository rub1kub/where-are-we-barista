# КРОТ как расчётный модуль

Документ описывает, как MVP можно оформить как отдельную расчётную программу для гражданского сценария проверки положения по радиовысотомеру и карте высот.

## Роль модуля

КРОТ не управляет воздушным судном. Модуль принимает проверочный журнал радиовысотомера, постоянную высоту `1500 м MSL` и карту высот, затем выдаёт расчётную оценку положения и метрики качества.

```text
РВ AGL / NMEA ─────┐
1500 м MSL ────────┤
карта высот района ┤
                   ↓
        terrain_nav_service
                   ↓
 local_x_m / local_y_m + lat/lon + Vпут + азимут + достоверность
                   ↓
        отчёт проверки / операторский монитор
```

## Входы

| Вход | Формат | Частота | Назначение |
| --- | --- | ---: | --- |
| Радиовысотомер | NMEA-0183 GGA | 1-10 Гц | Расстояние от борта до поверхности, AGL |
| Постоянная высота | `1500 м MSL` | параметр кейса | Абсолютная высота борта |
| Карта высот | Copernicus GLO-30 / SRTM / ALOS | заранее загружена | Эталонный профиль рельефа |
| Параметры поиска | диапазон скорости, шаг, окно | перед расчётом | Ограничения корреляционного поиска |

## Выходы

| Поле | Что значит | Куда передаётся |
| --- | --- | --- |
| `local_x_m` / `local_y_m` | локальные координаты на карте высот, м | отчёт, монитор, журнал |
| `lat` / `lon` | оценка координат WGS-84 | отчёт, монитор, журнал |
| `ground_speed_mps` | путевая скорость | навигационный монитор |
| `azimuth_deg` | путевой угол | отчёт проверки |
| `confidence` | достоверность оценки | логика принятия решения |
| `uncertainty_m` | оценка радиуса ошибки | контроль безопасности |
| `navigation_status` | `FIX VALID`, `FIX DEGRADED`, `FIX AMBIGUOUS`, `LOW RELIEF`, `NO FIX` | операторский статус |
| `compute_ms` | время расчёта | оценка эффективности |

## Режим проверки

Основной режим для хакатона: КРОТ считает положение по журналу радиовысотомера и карте высот. Дополнительные датчики и truth-траектория не передаются в solver.

```json
{
  "local_x_m": 158820,
  "local_y_m": 54026,
  "lat": 60.829960,
  "lon": 105.488265,
  "ground_speed_mps": 44.0,
  "azimuth_deg": 73,
  "confidence": 0.90,
  "uncertainty_m": 42,
  "navigation_status": "FIX VALID",
  "compute_ms": 1600
}
```

`course_correction_deg` рассчитывается как разница между текущим оценённым азимутом и направлением на плановую линию маршрута через точку упреждения. Если статус `NO FIX`, `LOW RELIEF` или `FIX AMBIGUOUS`, поле остаётся `null`: система не выдаёт совет по курсу при ненадёжной навигационной оценке.

### Внешний источник навигационной оценки

Следующий этап: передавать оценку как внешний источник положения через serial, CAN, MAVLink или внутренний API бортовой системы. Конкретный транспорт выбирается под автопилот и требования сертификации.

## Что реализовано в MVP

- Парсинг NMEA-0183 радиовысотомера.
- Преобразование `terrain MSL = 1500 м MSL - RA AGL`.
- Корреляционный поиск по карте высот.
- Оценка координат, Vпут, азимута, совпадения, ошибки профиля и достоверности.
- Статусы качества навигации.
- Постфактум-анализ вставленного NMEA-журнала.
- Время расчёта как метрика эффективности.

## Что не является частью MVP

- Управление рулями, тягой и стабилизацией.
- Failsafe-логика полётного контроллера.
- Серийная интеграция с конкретным бортовым протоколом.
- Использование IMU, камеры, FPV, GPS/ГНСС-трека или других дополнительных датчиков.
